### Как отдать расширение другим пользователям
- Для распространения: опубликуйте расширение и задеплойте бэкенд с HTTPS и корректным CORS.

#### Варианты публикации
- Публикация в Chrome Web Store (рекомендуется)
  - Зарегистрируйте аккаунт разработчика (разовый взнос).
  - Подготовьте описание, скриншоты, иконки, политику конфиденциальности.
  - В `manifest.json` оставьте только необходимые разрешения; укажите продакшн-домен API в `host_permissions`.
  - Разместите бэкенд (см. ниже) на публичном HTTPS-домене и обновите `backendBaseUrl` (через `options.html` или дефолт в `background.js`).
  - В бэкенде ограничьте CORS до вашего `chrome-extension://<EXT_ID>` и продакшн-домена.

- Внутренняя раздача (без магазина)
  - Для команды: включать Developer Mode и ставить "Load unpacked".
  - Для компаний (GWS): развернуть через Admin Console как "Force-install" и указать ZIP/URL.
  - Учтите: обычные CRX-файлы вне Web Store для массовой установки блокируются.

- Бэкенд (обязательно)
  - Деплой FastAPI на любой PaaS с HTTPS: например, Fly.io / Render / Railway / Cloud Run.
  - Настройки:
    - ENV: `GEMINI_API_KEY`, `EXA_API_KEY`.
    - CORS: разрешить `chrome-extension://<EXT_ID>`, ваш домен.
    - TLS: только `https://...`.
  - В `manifest.json` обновить `host_permissions` под ваш домен API.
  - По возможности ограничить `"<all_urls>"`, если не требуется.
